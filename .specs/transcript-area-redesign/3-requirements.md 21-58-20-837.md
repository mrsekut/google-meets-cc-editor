# Requirements: TranscriptArea Redesign

## User Story

ミーティング参加者として、字幕ログの誤字を素早く修正し、全文をコピーしたい。
ミーティング中にキーボードだけで操作を完結させたい。

## Acceptance Criteria

- [ ] 字幕が確定するたびに、セグメント（speaker + time + text）がリストに追加される
- [ ] セグメントをクリックすると編集モードに入り、テキストを修正できる
- [ ] 編集モードで上下矢印キーを押すと、隣のセグメントに編集が移動する
- [ ] Enter で編集確定、Esc で取り消し
- [ ] コピーボタンで全文がクリップボードにコピーされ、視覚的フィードバックがある
- [ ] 新しいセグメント追加時、末尾に自動スクロールする（ユーザーが上にスクロール中は追従しない）
- [ ] TranscriptAreaコンポーネントは字幕ロジックに依存しない（汎用的な編集可能リスト）

## Scope

### In Scope

- contentEditable廃止 → セグメント単位のReactコンポーネント描画
- セグメントのインライン編集（クリック + キーボードナビゲーション）
- 全文コピーボタン（フィードバック付き）
- useCaptionObserverのシグネチャ変更（`transcriptRef` → `onAppendSegment`コールバック）

### Out of Scope

- セグメントの削除・並べ替え
- セグメントの結合・分割
- 編集履歴（undo/redo）
- テキスト検索・置換
